# 音频系统页面与组件设计说明（桌面端优先）

## 全局规范

### Layout
- 桌面端优先（≥1280px）：设置弹窗固定宽 520px，居中；页面主体沿用现有“顶部导航 + 主内容卡片”。
- 交互密度：音量滑条与开关集中在弹窗内，避免占用大厅/对局主信息区域。

### Meta Information（统一默认）
- title 模板：`{页面名} - 联机大厅`
- description：`支持音频设置、语音播报队列、预加载与中断恢复。`
- Open Graph：沿用现有规范。

### Global Styles（Design Tokens - 音频相关补充）
- Slider：高度 4px，Thumb 14px；hover 时轨道高亮；拖动中显示数值百分比。
- Toggle：静音开关采用 Secondary 风格；开启时使用 Primary。
- 音量数值：以 0–100 显示，内部映射到 0.0–1.0。
- Toast：播报/恢复提示走统一右上角 Toast；“被打断/未解锁”提示使用 Banner（更醒目但不挡操作）。

## 页面 1：大厅（/）

### Page Structure
- 顶部导航 NavBar + 主内容（房间列表）不变；在右侧操作区新增“音频”入口。

### Sections & Components
1) 顶部导航（音频入口）
- 右侧新增按钮：`音频`（或小齿轮图标 + 文案）。
- 状态：
  - 未解锁音频时：按钮旁显示小点提示“需点击启用声音”。

2) 音频首次解锁 Banner（仅在需要时出现）
- 位置：导航下方、内容区上方。
- 文案：`浏览器限制自动播放，请点击启用声音`。
- 操作：Primary 按钮 `启用声音`；Secondary `保持静音`。

3) 音频设置弹窗 AudioSettingsModal（全局组件，来自大厅入口打开）
- 结构：
  - Header：标题“音频设置” + 关闭按钮
  - Body：三级音量 + 静音 + 测试区 + 高级操作
  - Footer：`恢复默认`（Secondary）+ `关闭`（Primary）

- 三级音量区（核心）
  - Master：滑条 + 数值 + 快速静音图标
  - BGM：滑条 + 数值
  - SFX：滑条 + 数值（包含语音播报）
  - 交互：拖动实时生效；键盘左右键微调（±1）。

- 测试区
  - 按钮：`测试音效`（播放短 click/notify）
  - 按钮：`测试语音`（入队一个示例语音，验证队列）

- 高级操作（仍属必需能力的 UI 出口）
  - 按钮：`清空语音队列`（立即停止当前语音并清空）
  - 文案说明：`语音按顺序播报，高优先级会打断`（一行小字）

## 页面 2：房间/对局（/room/:roomId）

### Page Structure
- 顶部导航 RoomNav 保持；右侧同样提供“音频”入口复用 AudioSettingsModal。
- 主内容仍为席位面板 + 行动面板；音频相关信息以“轻提示”为主，避免喧宾夺主。

### Sections & Components
1) 事件播报提示 Toast（弱提示）
- 当语音任务入队：可选显示 `已播报：轮到你`（默认关闭或仅调试模式）。
- 当队列被打断：Toast `高优先级提示已插队播报`。

2) 中断恢复 Banner（关键状态）
- 切后台/系统中断导致 AudioContext 暂停时：显示 `声音已暂停，点击恢复`（Primary：恢复）。
- 恢复成功：Toast `声音已恢复`。

3) 重连后的播报摘要（对局页信息辅助）
- 重连成功时，在 RoomStatusBar 下方短暂显示摘要条（3–5s）：
  - `你在 X 号位｜当前轮到 Y｜剩余 Z 秒`
- 若同时触发语音：摘要条与语音一致，避免信息冲突。

## 交互状态与无障碍（必要细节）
- 所有音频按钮在“未解锁音频”时不报错：点击会触发解锁流程或提示。
- Slider 与按钮支持键盘操作与可见 focus ring。
- 弹窗打开时锁定背景滚动，Esc 关闭；移动端可后续适配为底部抽屉（非本阶段重点）。
